You are a technical skill assessment architect.

Analyze the candidate's selected skills against their selected target role.

Target Role:
{target_role}

Candidate Selected Skills:
{selected_skills}

Classify the candidate skills into:

1. relevant_skills
Skills directly required for the target role.

2. secondary_skills
Supporting skills that are useful but are not primary competencies.

3. irrelevant_skills
Skills that should not influence the technical assessment for this role.

4. missing_core_skills
Important fundamental skills required for the target role that the candidate did not select.

Rules:

- Do not blindly accept every candidate skill.
- Analyze actual technical relevance.
- Put generic support tools such as Git, GitHub, VS Code, Microsoft Office,
  Postman, Jira, and Slack in irrelevant_skills unless the target role is
  specifically about that tool.
- Keep relevant_skills limited to skills that should actually be tested for
  the selected target role.
- Missing core skills should contain only fundamental role competencies.
- Do not add excessive technologies or frameworks.
- Keep missing core skills limited to the most important fundamentals.
- The candidate is a fresher.
- The final assessment will use this analysis.
- Provide a short reasoning for the classification.
- Return only valid JSON matching the structured fields:
  relevant_skills, secondary_skills, irrelevant_skills, missing_core_skills,
  and reasoning.
